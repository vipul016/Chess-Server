# вҷҹпёҸ Real-Time Multiplayer Chess Platform

A production-style, server-authoritative multiplayer chess platform built with Node.js, WebSockets, PostgreSQL, and Stockfish. Supports matchmaking, bots, spectators, Elo ratings, authenticated reconnection, and asynchronous game analysis.

## вңЁ Features

**Real-time gameplay**
- Authoritative move validation over WebSockets using `chess.js` вҖ” all game rules (legality, turns, checkmate/stalemate/draw detection) enforced server-side
- Server-owned chess clocks with automatic timeout (flag-fall) detection via a background sweeper
- Resignation and draw offer/response flow

**Matchmaking & User Profiles**
- Public user profiles showcasing Elo ratings, Win/Loss/Draw records, bios, and recent match histories
- Global matchmaking queue and private room generation via shareable 6-character codes
- Robust disconnection handling вҖ” players can refresh or disconnect and seamlessly re-join their active game seamlessly using their JWT session token
- Live spectator mode for any ongoing game

**Stockfish Integration**
- Play against AI bots of configurable difficulties
- Asynchronous, move-by-move post-game analysis generated by a dedicated Stockfish subprocess and custom in-memory task queue

## рҹӣ пёҸ Tech Stack

- **Runtime Engine:** Node.js (TypeScript)
- **Real-Time Communication:** WebSockets (`ws`)
- **Web Framework (REST):** Express.js
- **Database / ORM:** PostgreSQL managed via Prisma
- **Chess Logic:** `chess.js` (validation), Stockfish 16 C++ Engine (bots & analysis)
- **Deployment:** Docker, Render (Backend), Supabase Connection Pooler (DB)

## рҹ”Ң WebSocket Events

Websocket messages are transmitted as stringified JSON matching a `type` discriminant. 

**Client-to-Server Messages:**
- `find_match` - Enter the global matchmaking queue.
- `cancel_find_match` - Leave the matchmaking queue.
- `create_private_room` - Generate a unique 6-character room code.
- `join_private_room` - Join a private game via code.
- `play_bot` - Start a game against the Stockfish engine.
- `move` - Submit a chess move (Standard algebraic notation or from/to).
- `offer_draw` / `accept_draw` - Draw negotiation mechanics.
- `resign` - Immediately concede the game.
- `chat` - Send a message to the opponent.

**Server-to-Client Messages:**
- `match_found` - Both players are matched, game is initializing.
- `state` - The authoritative game state (FEN board, turn, remaining clock time).
- `game_over` - The game has concluded (Checkmate, Draw, Resignation, Timeout).
- `chat` - Incoming message from the opponent.
- `error` - Invalid move, unauthorized action, or server error.

## рҹ§Ә Testing Breakdown

The backend contains a comprehensive testing suite simulating the entire application flow without requiring a live database connection or manual browser.

- [x] **Auth:** Validates Express signup/login schemas, JWT minting, and duplicate user rejections.
- [x] **Gameplay:** Ensures illegal moves are rejected, turns are enforced, and clock boundaries hold.
- [x] **Matchmaking:** Validates exact-match and private room joining limits.
- [x] **Elo:** Unit tests mathematically verify winner/loser K-factor rating deltas.
- [x] **Chat:** Tests WebSocket broadcast isolation (messages only go to the specific room).
- [x] **Rooms:** Asserts memory cleanup when games conclude or participants permanently disconnect.

## рҹҡў Deployment

The backend is fully containerized and includes the Stockfish binary alongside the Node.js application.

### 1. Database Setup (Supabase / Neon)
The application requires a PostgreSQL database. If you use Supabase, **you must use the Transaction Connection Pooler URL** (port `6543`) because many hosting providers do not support Supabase's direct IPv6 connections.
- Ensure your connection URL appends `?pgbouncer=true&connection_limit=1`.

### 2. Backend Hosting (Render / Railway)
1. Import your `chess-server` repository to Render.com as a **Web Service**.
2. Render will automatically detect the provided `Dockerfile` and use the Docker runtime.
3. Configure the following Environment Variables in the Render dashboard:
   - `DATABASE_URL`: Your Supabase Connection Pooler URL.
   - `JWT_SECRET`: A secure random string for authentication tokens.
4. (Optional) The `Dockerfile` handles building the app and downloading Stockfish. The `package.json` start script will automatically push the database schema (`npx prisma db push`) upon startup, bypassing any local network database port blocks.
